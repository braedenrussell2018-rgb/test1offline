# Acceptable Use Policy

**Version:** 1.0  
**Last Updated:** January 2026  
**Classification:** Public

## 1. Purpose

This Acceptable Use Policy (AUP) defines the acceptable and prohibited uses of Serial Stock Suite and its associated services. By using the Software, users agree to comply with this policy.

## 2. Scope

This policy applies to:
- All users of Serial Stock Suite
- All data entered, stored, or processed through the Software
- All integrations and connected third-party services
- All devices used to access the Software

## 3. Acceptable Uses

### 3.1 Authorized Activities

Users MAY use Serial Stock Suite for:

- **Inventory Management:** Tracking, managing, and reporting on inventory items
- **Customer Relationship Management:** Managing customer contacts, communications, and transactions
- **Financial Operations:** Creating invoices, quotes, tracking expenses, and financial reporting
- **Vendor Management:** Managing vendor relationships and purchase orders
- **Team Collaboration:** Sharing business data with authorized team members
- **Third-Party Integration:** Connecting to approved services (e.g., QuickBooks) for business operations
- **AI-Assisted Analysis:** Using AI features for business insights and conversation analysis

### 3.2 User Responsibilities

Users MUST:

- Maintain confidentiality of their account credentials
- Use strong, unique passwords
- Enable multi-factor authentication when available
- Report suspicious activity immediately
- Keep contact information current
- Use the Software only for lawful business purposes
- Respect intellectual property rights
- Comply with all applicable laws and regulations

## 4. Prohibited Uses

### 4.1 Security Violations

Users MUST NOT:

- Share login credentials with unauthorized individuals
- Attempt to access accounts or data belonging to others
- Circumvent, disable, or interfere with security controls
- Use automated tools to access the Software without authorization
- Probe, scan, or test system vulnerabilities
- Attempt to reverse engineer or decompile the Software
- Interfere with service availability for other users

### 4.2 Data Misuse

Users MUST NOT:

- Store illegal or unlawful content
- Upload malicious software, viruses, or harmful code
- Process or store payment card data in violation of PCI-DSS
- Collect personal data without proper consent
- Transfer data to unauthorized parties
- Use data for purposes beyond authorized business operations
- Export data in violation of export control laws

### 4.3 Fraudulent Activities

Users MUST NOT:

- Misrepresent identity or affiliation
- Create false or misleading invoices
- Manipulate financial records for fraudulent purposes
- Use the Software for money laundering or financial crimes
- Infringe on trademarks, copyrights, or other intellectual property

### 4.4 Harmful Content

Users MUST NOT store or transmit:

- Content that is defamatory, obscene, or threatening
- Content that promotes discrimination or hatred
- Content that violates the rights of others
- Spam or unsolicited commercial communications
- Content that violates applicable laws

## 5. Account Security

### 5.1 Password Requirements

- Minimum 12 characters
- Combination of uppercase, lowercase, numbers, and symbols
- No commonly used passwords or dictionary words
- Unique password not used on other services
- Changed every 90 days

### 5.2 Multi-Factor Authentication

- Required for administrative accounts
- Recommended for all user accounts
- Approved methods: Authenticator apps, hardware tokens
- Backup codes must be stored securely

### 5.3 Session Management

- Sessions timeout after 30 minutes of inactivity
- Users should log out when finished
- Active sessions reviewed periodically
- Unusual session activity triggers security alerts

## 6. Data Handling

### 6.1 Personal Data

When handling personal data, users must:

- Collect only necessary information
- Obtain appropriate consent when required
- Keep data accurate and up-to-date
- Delete data when no longer needed
- Report data breaches immediately

### 6.2 Confidential Information

When handling confidential business data:

- Access only what is necessary for your role
- Do not share with unauthorized parties
- Use secure channels for transmission
- Store only in approved locations
- Follow data classification guidelines

### 6.3 Third-Party Data

When integrating with third-party services:

- Only connect authorized services
- Review permissions before granting access
- Monitor data shared with integrations
- Disconnect unused integrations promptly
- Comply with third-party terms of service

## 7. Resource Usage

### 7.1 Fair Usage

Users should:

- Use resources efficiently and responsibly
- Avoid activities that degrade performance for others
- Report system issues or performance problems
- Not monopolize system resources

### 7.2 API Usage

When using APIs:

- Follow rate limits and quotas
- Use appropriate authentication
- Cache responses when practical
- Report API issues promptly

## 8. Monitoring and Enforcement

### 8.1 Activity Monitoring

The Company may monitor:

- Login attempts and authentication events
- Data access patterns
- System resource usage
- Integration activities
- Security-relevant events

### 8.2 Audit Logging

The following activities are logged:

- Account creation and modifications
- Role and permission changes
- Data exports
- Administrative actions
- Security events

### 8.3 Enforcement Actions

Violations may result in:

| Severity | Action |
|----------|--------|
| Minor | Warning and education |
| Moderate | Temporary access restriction |
| Serious | Account suspension |
| Severe | Account termination and legal action |
